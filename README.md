# QNAssistant / 千牛话术助手

> 一个本地运行的千牛客服话术悬浮工具：分组管理、点击上屏、吸附跟随、最小化联动、悬浮图标恢复，都围绕“少切窗口、少重复输入”来做。

## 隐私说明

QNAssistant 默认只在本机运行，不依赖账号体系，也不需要连接云端服务。

- 不收集、不上传、不同步任何用户数据。
- 不包含统计、埋点、登录、远程控制或外部接口调用。
- 用户自定义话术只保存在程序同目录下的 `phrases.txt`。
- 诊断文件 `qn_uia_probe.txt` 仅在本地生成，用于排查输入框定位问题，不会自动上传。
- 工具不会主动保存或上传千牛聊天记录、客户信息、订单信息或店铺数据。
- 请不要在 `phrases.txt` 中保存账号密码、客户隐私或其他敏感信息。

## 项目定位

千牛桌面端本身没有一个稳定、公开、易调用的话术注入接口。这个项目的目标不是“深度接管千牛”，而是提供一个尽量轻、尽量稳的本地辅助层：

- 贴在千牛右侧，减少来回切换；
- 把高频回复沉淀成文本话术；
- 在不侵入千牛客户端的前提下，尽量实现一键上屏；
- 遇到千牛版本差异时，保留诊断信息，便于继续定位。

## 当前功能

### 话术管理
- 话术保存在工具根目录 `phrases.txt`
- 支持分组
- 支持自定义分组名称
- 支持新增、编辑、删除分组和话术
- 支持搜索话术
- 支持刷新重载话术文件
- 仅复制 `QianniuAssistant.exe` 到新目录时，会自动创建预置 `phrases.txt`

### 悬浮窗与吸附
- 无边框悬浮窗 UI
- 自动吸附在千牛接待台右侧
- 吸附模式下允许窗口超出屏幕范围，始终优先贴住千牛右边缘
- 跟随刷新频率已提升，拖动千牛时跟随更紧
- 支持手动拖动窗口
- 双击标题栏可切换自动吸附 / 手动模式

### 最小化与恢复
- 点击最小化后可切换为圆形悬浮图标
- 悬浮图标支持拖动
- 千牛手动最小化时，工具会联动隐藏主窗口并保留悬浮图标
- 点击悬浮图标时，可同时恢复千牛和工具窗口
- 恢复后会重新吸附到千牛右侧

### 上屏能力
- 点击话术后优先尝试直接注入到千牛输入区域
- 当前实现包含多级回退链路：
  1. UI Automation `ValuePattern`
  2. UI Automation `LegacyIAccessible`
  3. 独立输入控件句柄上的 `WM_PASTE` / `EM_REPLACESEL`
  4. 聚焦后 `Ctrl+V`
- 当千牛控件结构不可直接访问时，会尽量降级处理，并保留诊断文件供排查

## 为什么上屏不是 100% 保证

千牛桌面端是 **Qt + CEF** 架构，不同版本、不同页面形态下，输入区可能是：

- 标准可访问编辑控件；
- Qt 自绘控件；
- CEF 内部富文本编辑器；
- 没有独立窗口句柄的黑盒区域。

这意味着“自动上屏”只能做成**尽量兼容**，而不是对所有千牛版本都 100% 保证。README 里把这一点写清楚，比硬吹稳定更靠谱。

## 快速开始

1. 打开千牛接待台。
2. 运行 `QianniuAssistant.exe`。
3. 首次运行如果同目录没有 `phrases.txt`，程序会自动创建一份预置话术。
4. 工具启动后会自动尝试吸附到千牛右侧。
5. 点击话术卡片，程序会尝试把对应内容上屏到千牛输入区域。
6. 如果本次没有成功上屏，文本通常仍会保留在剪贴板，可先手动粘贴，再结合 `qn_uia_probe.txt` 排查。

## 常用操作

| 操作 | 说明 |
|---|---|
| 拖动标题栏 | 切换到手动模式并移动窗口 |
| 双击标题栏 | 切换自动吸附 / 手动模式 |
| 点击最小化按钮 | 收起为圆形悬浮图标 |
| 点击圆形图标 | 恢复千牛与工具窗口，并重新吸附 |
| `F5` | 重新加载 `phrases.txt` |
| `Esc` | 退出工具 |

## phrases.txt 格式

```txt
[问候语]
欢迎光临 = 亲，欢迎光临！有什么可以帮您的吗？
感谢咨询 = 感谢咨询，祝您生活愉快！

[物流相关]
发货时间 = 亲，我们会尽快为您安排发货哦~
物流查询 = 亲，这边帮您查询一下物流进度，请稍等。
```

说明：

- `[分组名]` 表示一个分组
- `标题 = 话术内容` 表示一条话术
- 也可以只写一行纯文本，程序会自动截取前 20 个字作为标题
- 修改文件后，回到工具内按 `F5` 或点击“刷新”即可重新加载

## 诊断文件

如果点击话术后仍然无法稳定上屏，工具会在程序目录下生成：

```txt
qn_uia_probe.txt
```

这个文件用于记录当前千牛聊天窗口下命中的 UI Automation 候选控件信息，例如：

- score
- control type
- native hwnd
- class name
- automation id
- bounding rect

它的用途是帮助确认：

- 千牛输入框是否暴露为可访问控件
- 是否存在独立输入框句柄
- 当前命中的候选项到底是不是输入区域

如果后续继续排查，上这个文件比只说“还是不行”更有效。

## 从源码运行

### 环境依赖
- Windows
- Python 3.10+
- `pywin32`
- `comtypes`
- `pyinstaller`（用于打包）

### 安装依赖

```bash
pip install pywin32 comtypes pyinstaller
```

### 运行

```bash
python main.py
```

### 打包

```bash
pyinstaller build.spec
```

默认输出：

```txt
dist/QianniuAssistant.exe
```

## 已知限制

- 仅支持 Windows 桌面端千牛使用场景。
- 千牛版本变化可能导致输入框定位策略失效。
- 如果千牛当前输入区是完全黑盒控件，自动上屏可能只能退回剪贴板粘贴方案。
- 本项目不是阿里官方工具，与千牛官方没有隶属关系。

## 适合谁

- 有固定客服回复模板的店铺客服
- 希望把常用回复沉淀为本地文本话术的人
- 不想依赖 SaaS、账号同步或联网服务的人
- 需要一个可以直接打包带走的轻量桌面工具的人

## 后续方向

当前版本已经把重点放在三个方向：

1. 更顺手的悬浮窗交互
2. 更稳的吸附 / 最小化联动
3. 更透明的输入框诊断与上屏回退链路

如果后面继续迭代，优先级仍会围绕这三件事，而不是盲目堆功能。
